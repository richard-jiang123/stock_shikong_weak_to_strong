# 弱转强 (Weak-to-Strong) 量化交易系统

A-share market quantitative trading system based on the "弱转强" (Weak-to-Strong) pattern, featuring local SQLite caching, incremental data updates, and automated daily screening.

## 策略概述

**弱转强** 是一种经典的技术分析交易模式，核心逻辑：

> 股票经历一波强劲上涨 → 进入温和缩量回调（不破位） → 某日出现放量反转信号 → 弱转强买点确认

### 交易逻辑

```
第一波强势上涨 (连续3+天上涨, 涨幅≥15%)
         ↓
    温和回调整理 (回调≤20%, 持续3-15天, 阴线占比<70%)
         ↓
  弱转强反转信号 (放量阳线, 突破前日阴线)
         ↓
     买入 → 止损/止盈管理
```

### 四种信号类型

| 信号 | 含义 | 特征 |
|---|---|---|
| 异动不跌 | 前日大幅波动次日不跌反涨 | 振幅>6%, 次日涨幅>1% |
| 阳包阴 | 阳线完全包住前日阴线实体 | 涨幅>2%, 收盘>前日开盘 |
| 大阳反转 | 阴线后出现强势反转阳线 | 涨幅>3%, 前日收阴 |
| 烂板次日 | 涨停板未封住次日依然强势 | 前日涨幅>8%未封板, 次日涨幅>2% |

### 评分体系 (满分 ~80 分)

| 维度 | 权重 | 条件 |
|---|---|---|
| 一波涨幅 | 10-20分 | >30% 加20分, >20% 加10分 |
| 回调深度 | 10-15分 | <8% 加15分, <15% 加10分 |
| 当日涨幅 | 5-15分 | >7% 加15分, >5% 加10分, >3% 加5分 |
| 量比 | 5-10分 | >2x 加10分, >1.5x 加5分 |
| 均线排列 | 5-10分 | 多头排列(MA5>MA10>MA20) 加10分 |
| 信号类型 | 10分 | 异动不跌 额外加10分 |
| 基础分 | 5分 | 所有信号 |

### 风险管理

- **止损位**: 回调最低点 × 0.98 (2% 安全边际)
- **移动止盈**: 从最高点回撤 8% 且累计盈利 > 10% 时触发
- **最大持仓**: 20 个交易日强制平仓
- **板块联动**: 检查所属板块动量，强势板块信号权重更高


### 退出策略详解

系统采用四重退出机制，按优先级依次判断：

| 退出类型 | 笔数 | 占比 | 胜率 | 平均收益 |
|---|---|---|---|---|
| 移动止盈 | 3,448 | 54.1% | **86.1%** | **+11.02%** |
| 固定止损 | 1,224 | 19.2% | 5.6% | -10.88% |
| 时间止损 | 884 | 13.9% | 35.1% | -2.49% |
| 最终平仓 | 813 | 12.8% | 56.8% | +5.28% |

#### 1. 移动止盈 (Trailing Stop) — 核心盈利来源

**触发条件**: 从最高盈利点回撤 8%，且累计盈利已超过 10%

**逻辑**: 当持仓期间股价持续上涨，系统会记录最高点。一旦从最高点回撤超过 8%，
且整体盈利已超过 10%，立即出场。这样既锁定了大部分利润，又给了趋势继续发展的空间。

**效果**: 占总交易的 54.1%，胜率 86.1%，平均获利 +11.02%，是策略最主要的盈利来源。

**为什么有效**: 弱转强策略买入后如果趋势成立，通常会有一段快速拉升。
移动止盈能让利润"奔跑"到极限附近再锁定，避免过早止盈错过主升浪。

#### 2. 固定止损 (Stop Loss) — 刚性风控底线

**触发条件**: 盘中价格跌破回调最低点 × 0.98（留 2% 安全边际）

**逻辑**: 买入时就设定明确的止损位。止损位设在前期回调的最低点下方 2%，
如果跌破这个位置，说明"弱转强"判断失效——回调不是温和整理而是趋势反转。

**效果**: 占 19.2%，胜率仅 5.6%（因为触发的都是真亏损），但控制了单笔最大损失在 -10.88%。

**为什么必要**: 任何策略都有判断错误的时候。固定止损是最后的防线，
防止单次交易出现灾难性亏损。没有止损的策略，一次大亏可能吃掉十次盈利。

#### 3. 时间止损 (Time Stop) — 机会成本管理

**触发条件**: 持仓超过 5 个交易日后，若股价表现疲弱（涨幅不大或为负），可提前出场

**逻辑**: 弱转强信号如果有效，通常会在 3-5 个交易日内快速反应。
如果持仓 5 天以上还没涨起来，说明信号力度不足，继续持有只是浪费时间。

**效果**: 占 13.9%，胜率 35.1%，平均亏损 -2.49%。损失很小，主要是时间成本。

**为什么有效**: 避免了"等解套"的心理陷阱。时间止损帮你承认"这笔交易可能错了"，
把资金释放给下一个更好的机会。

#### 4. 最终平仓 (Final Exit) — 到期交割

**触发条件**: 回测区间到期，或数据结束时的自然平仓

**逻辑**: 当回测时间窗口到达终点，无论持仓盈亏，按收盘价强制平仓。
这不代表策略行为，而是回测统计口径的"到期交割"。

**效果**: 占 12.8%，胜率 56.8%，平均收益 +5.28%。数据表现中性偏正。

**实际交易中的含义**: 在实盘中，这对应于超过最大持仓天数（20天）后强制平仓。
如果一只股票 20 天还没触发止盈或止损，说明它处于"不上不下"的尴尬状态，
此时平仓是最安全的选择。

## 设计要点

### 数据层

- **WAL 模式**: SQLite Write-Ahead Logging，支持并发读写
- **增量更新**: 记录每只股票最后更新日期，只拉新数据
- **批量查询**: `get_kline_batch()` 使用一条 SQL IN 查询替代逐只查询
- **自动补全**: 扫描时发现数据不足，自动触发增量更新

### 策略层

- **ST 股过滤**: 自动剔除名称含 "ST" 的股票
- **板块动量**: 计算所属板块 10 日平均收益率，强势板块加分
- **多信号融合**: 不依赖单一技术指标，综合形态、量能、均线

### 风控层

- **固定止损**: 入场价跌破回调最低点 2% 以下立即止损
- **移动止盈**: 盈利后锁定利润，回撤 8% 出场
- **时间止损**: 20 个交易日未盈利强制平仓
